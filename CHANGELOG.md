# 更新日志

## 1.1.12

- 媒体 URL 下载分阶段超时（DNS 2s / 连接 3s / 读取 5s），流式读取并限制最大 5MB；已解析公网 IP 降低 DNS 重绑定。

## 1.1.11

- 加固媒体 URL 下载（SSRF：拒绝 localhost / 私有网段等）。
- ``_manifest.json`` 的 ``host_application.max_version`` 放宽为 ``1.99.99``。
- 词库编辑器默认仅监听 ``127.0.0.1``。

## 1.1.10

- 词库编辑器新增「合并重复词条」：回复与群策略相同的词条合并为主触发词 + 别名；写入前自动备份 ``keywords.json.bak_merge_*``。
- 合并转发消息（``raw_message`` 含 ``forward`` / 正文 ``【合并转发消息:…】``）整条不参与关键词与检测词匹配，避免 QQ 名误触发。
- 自动回复命中时输出 DEBUG 诊断日志（``processed_plain_text``、``raw_seg_types``、精简 ``raw_message``）。

## 1.1.9

- 修复 ``reply.respond_to_triggers_in_quote = false`` 时仍可能命中引用正文的问题：仅当消息带真实 ``reply`` / ``reply_to`` 时只匹配用户自己输入的文本；无真实引用时整段正文原样匹配。
- 词库编辑器「全选」：未选→全选，已全选→取消，部分选中→反选。

## 1.1.8

- 修复含空格触发词无法匹配的问题（不再只用消息首词比对）。
- 词条支持多别名 ``aliases[]``：编辑器「添加别名」逐条增加；命令 ``/添加关键词别名``、``/删除关键词别名``（检测词同理）。
- 词库编辑器新增「创建副本」：复制回复与群策略，触发词留空，填写并保存后才写入。
- 新增配置 ``reply.respond_to_triggers_in_quote``（默认 ``false``）：触发词仅出现在引用正文内时默认不响应。

## 1.1.7

- 回滚从入站消息 / 引用消息拉取视频的全部尝试（不再下载、不再写入 ``media_cache/videos``），避免无效性能开销。
- 视频仅支持本地加载：将文件放入 ``videos/``，经编辑器或 ``keywords.json`` 引用后发送。

## 1.1.6

- 诊断：当入站视频已被适配器收成纯文本占位（``segments=text`` / ``[视频] 文件: …``）时输出明确警告；MaiBot 核心现支持 ``type:video``→``FileComponent``（需适配器透传 url，不能只发占位文本）。

## 1.1.5

- 修复视频缓存：支持 NapCat 把本地路径塞进 ``url/file`` 的情况；补齐 MaiBot 可见的媒体缓存日志（白名单跳过 / 下载失败 / 已写入路径），便于排查为何未进入 ``media_cache/videos/``。
- 避免同一视频段重复下载落盘。

## 1.1.4

- 词库编辑器：图片/语音/表情/视频路径输入改为固定展示目录前缀（``images/`` 等），用户只需填写文件名。

## 1.1.3

- 修复视频被 MaiBot 归一成 ``type:file``（``name/size/url``）后无法写入 ``media_cache``、触发回复只剩「文件: xxx.mp4，大小: …」文本的问题；入库时剥离该类文件占位文案。

## 1.1.2

- 支持视频消息：引用视频后添加关键词，流程与图片相同（``before_process`` 写入 ``media_cache/videos/``，入库时提升到 ``videos/``）；发送使用 NapCat/OneBot ``video`` 段（``base64://``）。

## 1.1.1

- 修复引用图片后发送 `/添加关键词 <词>`（关键词后无正文）时，把整句命令误存为回复、触发后复读命令的问题。
- 修复同条命令消息附图时媒体缓存未合并的问题。
- 入站媒体缓存改写入独立目录 ``media_cache/``；词库永久目录 ``images/`` ``records/`` ``emojis/`` 保存后不再自动清理（此前未引用文件会在 save 时被删掉，导致引用图添加后消失）。
- ``media_cache/`` 仅在总文件数超过 100 时按修改时间滚动删除最旧文件；写入词库前会把缓存媒体提升到永久目录。

## 1.1.0

- 词条新增 `require_at_bot`（需 @ 机器人才触发）与回复 `weight`（多回复按权重抽取）。
- 新增命令：`/设置关键词需@`、`/设置关键词权重`（检测词同理）、`/重载词库`。
- 新增外部词库编辑器 `editor/`（总览、筛选、搜索、批量增删改）。

## 1.0.11

- 插件 ID 改为 `maibot_plugin.keywords_reply`，数据目录同步为 `data/plugins/maibot_plugin.keywords_reply/`（原为 `foolllll.keywords-reply`）。升级后请手动迁移旧数据目录。

## 1.0.10

- 新增 `/添加音乐 <关键词> <歌曲ID> [平台]` 命令，可直接用 NapCat 解析出的歌曲 ID 创建音乐卡片关键词回复；平台默认为 `163`（网易云）。

## 1.0.9

- 支持网易云/QQ 等 OneBot 音乐卡片：引用音乐消息添加关键词时保存 ``platform + id``，触发回复时重新发送标准音乐卡片，而非 ``[网易云音乐] 歌名`` 文本。

## 1.0.8

- 入站媒体缓存改为仅对白名单群聊生效（`[media_cache] group_whitelist`），默认不缓存，降低性能开销。

## 1.0.7

- 修复引用语音消息后执行 `/添加关键词` 提示「回复内容不能为空」：在 `before_process` 缓存每条入站语音/图片/表情二进制；引用导入时优先合并缓存，并增强语音段解析（`content` 字段、`dict` 嵌套、`record` 别名）。

## 1.0.6

- 修复 At 回复重复：避免 `before_process` 缓存与命令消息二次抓取叠加；文本中的 `@昵称` 在已有真实 At 时自动剥离。

## 1.0.5

- 纯插件方案：在 `chat.receive.before_process` Hook 中于 VLM 识图前缓存管理命令消息的富媒体二进制。
- 管理命令正则去掉 `^` 锚点，支持引用消息后发命令（如引用图片 + `/添加关键词 test`）。
- 说明：无法修改 MaiBot 主程序时，VLM 仍可能消耗 token，但图片可正确保存；可在 MaiBot 配置中关闭 VLM 以彻底避免识图。

## 1.0.4

- ~~配合 MaiBot 主程序「插件命令快路径」~~（需改主程序，已改为 1.0.5 纯插件方案）

## 1.0.3

- 添加图片回复时自动剥离 MaiBot VLM 生成的 `[图片：...]` 等占位描述，只保存真实图片/语音/表情媒体。

## 1.0.2

- 修复图片回复无法保存：命令消息现携带媒体二进制；`get_by_id` 返回值解包；支持从图片 URL 回退下载。
- MaiBot 主程序：引用消息发命令时自动剥离 `[回复了...]` 前缀，避免命令无法识别。

## 1.0.1

- 修复自动回复不生效：MaiBot 主链默认未启用 `ON_MESSAGE` 事件，自动回复改挂 `chat.receive.after_process` Hook；保留 `ON_MESSAGE` 处理器作兼容备用。

## 1.0.0

- 从 AstrBot 插件 `astrbot_plugin_keywords_reply` 迁移到 MaiBot SDK 2.x。
- 支持指令触发（关键词）与自动监听（检测词）两种模式。
- 支持文本、图片、At、语音、表情包的组合回复，及正则匹配、群聊黑白名单、冷却、变量模板。
- 富媒体通过触发消息与引用消息捕获（base64），以本地文件形式持久化，数据保存在可外部编辑的 `keywords.json`。
- 已放弃的 AstrBot 专有能力：自动撤回、合并转发聊天记录导入、WebUI 面板（详见 README）。
