# 词库外部编辑器

在浏览器中管理 `keywords.json`：总览、筛选、搜索，以及批量增删改。

## 启动

在仓库根目录执行（将 `--data-dir` 换成你的 MaiBot 插件数据目录）：

```bash
python editor/server.py --data-dir ".../MaiBot/data/plugins/maibot_plugin.keywords_reply"
```

或者直接启动 `editor.bat`（仅本机）。

默认监听 `http://127.0.0.1:8765`。

### 局域网其它设备访问

工作机性能不足时，可在本机启动服务，用手机 / 另一台电脑的浏览器编辑：

```bash
python editor/server.py --data-dir ".../MaiBot/data/plugins/maibot_plugin.keywords_reply" --host 0.0.0.0 --port 8765
```

或双击 `editor-lan.bat`。启动后终端会打印类似：

```text
http://192.168.x.x:8765
```

在同一局域网的其它设备浏览器打开该地址即可。

注意：

- 编辑器**没有登录鉴权**，仅在可信局域网使用。
- Windows 若打不开：防火墙放行入站 TCP `8765`（专用网络即可）。
- 数据仍写在工作机上的 `keywords.json`；保存后在 MaiBot 执行 `/重载词库`。

## 功能

- **总览**：关键词 / 检测词分页签切换，统计数量
- **筛选**：启用状态、需@、正则、群号、全文搜索（找关键词）；编辑弹窗内可按回复类型筛子词条
- **批量操作**：启用、禁用、开启/关闭需@、**批量群策略**、删除
- **单条编辑**：触发词、别名、群黑白名单、结构化回复行（文本 / 音乐 / 图片 / 语音 / 表情 + 概率%）
- **创建副本**：复制回复与群策略，触发词留空，填写后确定才写入
- **导入 / 导出**：JSON 文件

## 大词库性能

7000+ 词条时编辑器采用：

- **列表分页**（默认每页 50，可调 30/100/200）
- **搜索防抖**（约 280ms，避免每键重绘）
- **回复列摘要**（显示条数 + 首条预览，不再展开全部回复）
- **编辑弹窗内回复分页**（每页 40 条）

首次加载超大 JSON 仍可能需要数秒；加载完成后翻页、筛选会流畅很多。建议用搜索缩小范围后再编辑。  
若工作机浏览器卡顿，优先用「局域网其它设备访问」在性能更好的设备上打开页面。

## 回复编辑（结构化）

编辑词条时，**每条带边框的回复 = 一个 `entry`**（权重 + 概率%）。

其内按 **消息组** 分层，与 `keywords.json` 的 `parts[]` / `segments[]` 一致：

- **消息组**（标题「消息 1」「消息 2」…）= 触发后发送的**一条聊天消息**（对应 `parts[i]`）
- 组内每一行 = 该消息里的一个 segment（文本 / QQ表情 / 图片 / 语音 / 表情文件 / 音乐），合并到同一条 hybrid 消息发送
- **文本换行**：输入框内显示字面量 `\r\n` / `\n`（与 JSON 文件一致）；保存时自动还原为真实换行
- 行末 **+** 只在**当前消息组内**追加 segment；底部 **+ 添加下一条消息** 对应 OhData 的 `&` 分隔
- **删除消息** 移除整组；**×** 只移除组内一行
- 长按 **⠿** 仅在**同一消息组内**拖动排序

主列表用左侧搜索找关键词即可。

## 批量群策略

勾选多个词条 → **批量群策略**：

- **替换为以下策略**：统一设置白/黑名单与群号
- **在现有群号上追加**：保留原群号并追加新群号
- **设为全局启用**：黑名单模式且群列表为空
- **设为全局禁用**：关闭该词条

## 与 MaiBot 同步

编辑器直接写入 `keywords.json`。保存后请在群聊执行 **`/重载词库`**，或重启 MaiBot，插件才会读取最新数据。
