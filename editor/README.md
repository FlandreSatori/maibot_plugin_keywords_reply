# 词库外部编辑器

在本机浏览器中管理 `keywords.json`：总览、筛选、搜索，以及批量增删改。

## 启动

在仓库根目录执行（将 `--data-dir` 换成你的 MaiBot 插件数据目录）：

```bash
python editor/server.py --data-dir ".../MaiBot/data/plugins/maibot_plugin.keywords_reply"
```

或者直接启动 `editor.bat`

默认监听 `http://127.0.0.1:8765`，可以用``--host 0.0.0.0``将网页暴露到局域网，会把可用的访问地址贴到控制台。

### 访问密码（推荐在非本机访问时开启）

启动时加 ``--token``：

```bash
python editor/server.py --data-dir "..." --host 0.0.0.0 --port 8765 --token "你的密码"
```

然后用带查询参数的地址打开：

```text
http://127.0.0.1:8765/?token=你的密码
http://局域网IP:8765/?token=你的密码
```

校验通过后会写入 Cookie，同一浏览器后续打开与 API 请求无需反复带参数。未设置 ``--token`` 时行为与以前相同（无密码）。

> ⚠️注意⚠️，主动将后端暴露到公网视为已知潜在风险，普通用户请不要随意更改启动脚本；若必须局域网访问，请务必设置 ``--token``。

保存后请执行 `/重载词库`。

## 功能

- **总览**：关键词 / 检测词分页签切换，统计数量
- **筛选**：启用状态、需@、正则、群号、全文搜索（找关键词）；编辑弹窗内可按回复类型筛子词条
- **批量操作**：启用、禁用、开启/关闭需@、群策略、删除
- **单条编辑**：触发词、别名列表（逐条添加）、群黑白名单、结构化回复行（文本 / 音乐 / 图片 / 语音 / 表情 + 概率%）
- **创建副本**：复制回复与群策略，触发词与别名留空，填写后确定才写入
- **合并重复词条**：把回复与群策略相同的词条合并为「主触发词 + 别名」；先自动备份 ``keywords.json.bak_merge_*``
- **导入 / 导出**：JSON 文件

## 大词库性能

- **列表分页**（默认每页 50，可调 30/100/200）
- **搜索防抖**（约 280ms，避免每键重绘）
- **回复列摘要**（显示条数 + 首条预览，不再展开全部回复）
- **编辑弹窗内回复分页**（每页 40 条）

首次加载超大 JSON 仍可能需要数秒

## 回复编辑

**每条带边框的回复 （权重 + 概率%）是一个消息组：**

- **消息组**（消息1「消息段 1」「消息段 2」… ；消息2「消息段 1」「消息段 2」…）= 触发后发送的**一组聊天消息**，每组消息可以包含多条消息，按顺序发送。

- 一条消息里的每行 = 一段内容（文本 / QQ表情 / 图片 / 语音 / 表情文件 / 音乐），会合并到同一条消息发送

  >  文本输入框内的`\r\n` / `\n`在保存时会自动还原为真实换行

- 行末 加号**+** ：追加一段内容

- 长按 **⠿** 仅在**同一消息组内**拖动排序

## 批量群策略

勾选多个词条 → **批量群策略**：

- **替换为以下策略**：统一设置白/黑名单与群号
- **在现有群号上追加**：保留原群号并追加新群号
- **设为全局启用**：黑名单模式且群列表为空
- **设为全局禁用**：关闭该词条

## 与 MaiBot 同步

编辑器直接写入 `keywords.json`。保存后请在群聊执行 **`/重载词库`**，或重启 MaiBot，插件才会读取最新数据。

**⚠️注意⚠️浏览器缓存不代表词库实时数据，不要同时多人修改/多端修改！**
